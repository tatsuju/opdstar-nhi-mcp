What is this?

@opdstar/nhi-mcp lets any AI assistant compatible with Model Context Protocol — Claude Desktop, Cursor, ChatGPT Desktop, Continue, Zed, and more — directly query Taiwan's National Health Insurance (NHI) data curated and maintained by the OPDSTAR team.

Built on top of six months of structured-data curation from 健保署 publications, this package exposes four carefully-scoped read-only tools that let doctors and developers integrate real NHI lookup into their AI workflows with one command.

Four tools

lookup_rejection_code

Look up a 5-character NHI rejection code (e.g. 0317A, 0338A). Returns severity, category, and the official Chinese description.

get_procedures_for_icd

Given an ICD-10 code and specialty, return NHI procedure codes applicable — with points, audit notes, SOAP requirements.

get_indicator

NHI audit indicators (008, 014, 027, P043) — thresholds, applicable drugs/diagnoses, compliance meaning.

search_nhi_wiki

Semantic + full-text search across 9 categories of official 健保署 wiki content (8,232 chunks).

Quickstart for Claude Desktop

Add to ~/Library/Application Support/Claude/claude_desktop_config.json (Mac) or %APPDATA%\Claude\claude_desktop_config.json (Windows):

{
  "mcpServers": {
    "opdstar-nhi": {
      "command": "npx",
      "args": ["-y", "@opdstar/nhi-mcp"]
    }
  }
}

Restart Claude Desktop. You should see 4 new tools.

FAQ

What is Model Context Protocol (MCP)?

MCP is an open standard released by Anthropic in November 2024 that lets AI assistants call external tools through a standardized protocol. Think of it as "USB-C for AI agents" — write a server once, and any MCP-compatible client can talk to it.

Who should install this?

Taiwanese doctors using AI assistants for chart work, HIS and clinic software vendors integrating AI features, and researchers studying Taiwan's healthcare data.

Do I need an OPDSTAR account or API key?

No. Fully public and free under MIT license. No account, no API key, no registration. Rate limiting is based on IP only.

Is my patient data sent anywhere?

No. The tools are pure lookups over public NHI data. Only the code or diagnosis name you ask about is transmitted — never patient names, IDs, or clinical notes. The proxy does not log query content.

Is this an official NHI product?

No. Independently developed by the OPDSTAR team. Data is sourced from publicly available 健保署 publications. Not affiliated with or endorsed by Taiwan's NHI Administration.

Can I use this commercially?

Yes. MIT license permits unrestricted commercial use. You may embed @opdstar/nhi-mcp in paid products without royalties. We only ask that the powered_by attribution remains in tool responses.

How often is the data updated?

Monthly. The OPDSTAR team syncs structured tables and wiki chunks when 健保署 publishes new rules.
