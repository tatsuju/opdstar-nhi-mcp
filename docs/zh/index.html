這是什麼？

@opdstar/nhi-mcp 讓任何支援 Model Context Protocol 的 AI 助理（Claude Desktop、Cursor、ChatGPT Desktop、Continue、Zed 等） 能直接查詢台灣健保資料 — 由 OPDSTAR 團隊 整理與維護。

這是累積六個月結構化整理健保署官方資料後的成果，封裝成四個精心設計的**唯讀查詢工具**， 讓台灣醫師與 HIS 廠商能一行指令就把健保資料整合進 AI 工作流。

適合誰用？

  • 🩺 台灣醫師 — 在 Claude Desktop / Cursor 邊寫病歷邊用 AI 查核刪風險、處置碼、健保指標
  • 🏥 HIS / 診所系統廠商 — 把健保資料整合進你的 AI 醫療工具，免爬蟲、免買斷
  • 📊 醫療 / 健保資料研究者 — 結構化資料可直接餵 AI 推理，取代手動翻健保署 PDF
  • ⚕️ 醫學教育 — 教學用例、學生查詢、住院醫師培訓輔助

實際情境

🩺 情境 1 — 開診間核刪預判
你在 Claude Desktop 邊打病歷邊問：
「我今天給 L30.9 慢性濕疹病人開了 prednisolone 30mg QD 5 天，會不會被核刪？」
Claude 自動呼叫 lookup_rejection_code + get_indicator：
「依健保審查 008 指標，口服類固醇 + 濕疹類診斷若 PE 描述不足，會觸發核刪代碼 0320A（處方超過審查標準用量）。建議：PE 補充病灶面積 ≥ 30% 全身，或加入更嚴重的 ICD-10 佐證。」
🔍 情境 2 — 處置碼速查
「中耳炎 H66.9 耳鼻喉科能申報哪些處置？」
Claude 呼叫 get_procedures_for_icd：
「27002C 耳鏡檢查（25 點）、27018C 耳膜穿刺（120 點）、36005C 鼓室通氣管置入（記錄雙側或單側）…」
完整清單 1,497 筆處置 × 20 專科。
📚 情境 3 — Wiki 條文查詢
「健保『慢性病連續處方箋』的規定原文？」
Claude 呼叫 search_nhi_wiki：
「慢性病連續處方箋每次調劑之藥品使用期間，最長以 3 個月為限。病情穩定且符合慢性病定義者方可開立…」
附上健保署原文 URL。

四個查詢工具

lookup_rejection_code 健保核刪代碼查詢

輸入 5 碼（如 0317A、0338A），回傳嚴重度、類別、官方說明。

get_procedures_for_icd ICD-10 對應處置碼

給 ICD-10 + 專科，列出該科適用處置、點數、審查注意事項、SOAP 要求。

get_indicator 健保審查指標

查 008 / 014 / 027 / P043 等指標的閾值、適用藥物、適用診斷。

search_nhi_wiki 健保署 Wiki 語意搜

語意 + 全文搜尋 9 大分類的官方 Wiki 內容（8,232 chunks）。

Claude Desktop 快速上手

找這個檔案：

  • Mac：~/Library/Application Support/Claude/claude_desktop_config.json
  • Windows：%APPDATA%\Claude\claude_desktop_config.json
  • Linux：~/.config/Claude/claude_desktop_config.json

加入以下設定：

{
  "mcpServers": {
    "opdstar-nhi": {
      "command": "npx",
      "args": ["-y", "@opdstar/nhi-mcp"]
    }
  }
}

重啟 Claude Desktop，左下角 ⚙ → Developer 確認 opdstar-nhi 狀態 running，就能開始問問題了。沒裝 Node.js 的話到 nodejs.org 裝 LTS 版即可。

常見問題

什麼是 Model Context Protocol (MCP)？

MCP 是 Anthropic 於 2024 年 11 月發佈的開放標準，讓 AI 助理能透過統一協定呼叫外部工具。類比：MCP 就像「AI agent 界的 USB-C」— 一端寫好 server，任何 MCP 相容 client（Claude Desktop、Cursor、ChatGPT Desktop）都能使用，無需客製整合。

我是醫師不懂技術，這個對我有什麼用？

你只要在 Claude Desktop 設定檔貼一段 10 行 JSON、重啟 App，就多了四個健保查詢工具。之後在 Claude 用自然語言問「0317A 核刪代碼什麼意思？」或「L30.9 皮膚科能申報哪些處置？」AI 會自動呼叫 tool 並回真實健保資料。你完全不需要懂程式。

健保 MCP 跟直接到健保署網站查有什麼差別？

差在整合性。健保署官網要手動點頁面、翻 PDF、切多個 tab 比對。這個 MCP 讓你在 AI 助理不離開當前工作流就能查資料，結構化的 JSON 回應也能被 AI 繼續推理（例如「這三個代碼的關係？」）。

不是取代健保署，是把健保署的資料包裝成 AI 能直接使用的結構化格式。

需要 OPDSTAR 帳號或 API key 嗎？

不用。完全公開免費，MIT 授權。不需要註冊、不需要 API key。proxy 層 (opdstar.com) 的 rate limit 是按 IP 計算，不追蹤身份。

這是健保署官方工具嗎？

不是。OPDSTAR 團隊獨立開發，所有資料整理自健保署公開資源。不是健保署官方合作單位，也沒有官方認證。最權威的資料仍以健保署官方公告為準。

我的病患資料會被上傳嗎？

完全不會。這四個 tool 只查公開健保代碼、處置、Wiki，不傳送任何病患名字、病歷號、臨床內容。當你問 AI「0317A 是什麼」時，MCP 只送「0317A」這個字串到 proxy。

proxy 層也不記錄查詢內容（僅統計匿名總量用於 rate limit）。

資料多久更新一次？

每月同步健保署最新公告。重要更新會發 GitHub Release + CHANGELOG 公告。到 repo 右上 Watch → All Activity 可訂閱變更通知。

我的診所或醫院可以拿去商業使用嗎？

可以。MIT 授權不限商業用途。你可以把 @opdstar/nhi-mcp 嵌進 HIS 系統、整合到付費醫療 AI 產品、內部員工訓練使用。唯一請求：保留 powered_by 的 OPDSTAR 標示（工具回應內建，不應被移除）。
